# UX Research Plan

## Objectives

Validate the map-first vertical slice and critical journeys before public release.

## Participants (target n=12–16)

- Younger first-time visitors
- Working-age family planners
- Older adults (55+)
- Low-digital-literacy smartphone users
- Partido residents and visitors unfamiliar with the area
- Screen-reader and keyboard-only users
- Low-bandwidth mobile users

## Methods

1. Moderated task tests (remote or in-person)
2. System Usability Scale (SUS) after core tasks
3. Expert heuristic review (see UX_RELEASE_CHECKLIST.md)

## Critical journeys

1. Select municipality from landing map or list
2. Open municipality profile
3. Search and open a place in Goa
4. Read verification and source on place page
5. Ask AI about the place
6. Create and save a day trip with budget estimate

## Success criteria

- ≥95% completion for municipality selection + profile open
- ≥90% overall unaided completion
- SUS ≥85
- No critical accessibility blockers on keyboard/list-only paths

## Schedule

- Round 1: prototype / staging vertical slice
- Round 2: integrated MVP candidate
- Round 3: pre-release confirmation

## Deliverables

- Session recordings (with consent)
- Issue log with severity and retest status
- USABILITY_TEST_REPORT.md summary
